# Home Assistant OS on Raspberry Pi 5

This directory contains the Home Assistant OS variant for building headless installation media.

## Features

- Automatically downloads latest Home Assistant OS for Raspberry Pi 5
- Pre-seeds network configuration (static IP or DHCP)
- Pre-populates initial Home Assistant onboarding
- Injects systemd service for first-boot status reporting
- Integrates with the status monitoring server (optional)

## Quick Start

```bash
./setup.sh haos
# Edit ../config.env with your settings
./build_and_flash.sh
```

In another terminal, monitor installation:

```bash
./broadcast.sh
```

## Configuration

Edit `../config.env` after running `./setup.sh haos`:

### Network

```env
# Static IP (leave empty for DHCP)
HA_STATIC_IP="192.168.1.42"
HA_GATEWAY="192.168.1.1"
HA_NAMESERVER="8.8.8.8"
# Or DHCP (leave HA_STATIC_IP empty)
HA_STATIC_IP=""
```

### Home Assistant Settings

```env
HOSTNAME="homeassistant"
HA_USERNAME="homeassistant"
PASSWORD_HASH="..."  # Generated by setup.sh

HA_LOCATION_NAME="Home"
HA_LATITUDE="40.7128"
HA_LONGITUDE="-74.0060"
HA_ELEVATION="0"
HA_UNIT_SYSTEM="metric"  # or "us_customary"
HA_TIMEZONE="UTC"
```

### Status Monitoring

```env
STATUS_IP="192.168.1.23"  # Your monitoring machine
STATUS_PORT="8081"
```

## Image Download

On first run, `./setup.sh haos` fetches the latest Home Assistant OS for RPi5 from GitHub:

```bash
./setup.sh haos                    # Downloads latest
./setup.sh haos --skip-download    # Use existing image
./setup.sh haos --download-only    # Download and exit
```

## Workflow

1. **Setup** — Initialize and download image

   ```bash
   ./setup.sh haos
   ```

   Edit `../config.env` with your preferences.

2. **Build** — Inject configuration and prepare for flashing

   ```bash
   ./build_and_flash.sh
   ```

   Prepares the final `.img` ready for micro-SD.

3. **Flash** — Use the root `build_and_flash.sh` to flash to USB/micro-SD

   ```bash
   cd ..
   ./build_and_flash.sh
   ```

4. **Monitor** (optional) — Watch installation progress

   ```bash
   ./broadcast.sh
   ```

## Technical Details

### Image Preparation

- Downloads latest `haos-rpi5-*.img.gz` from [Home Assistant OS releases](https://github.com/home-assistant/operating-system/releases)
- Decompresses to `.img`
- Mounts partitions (uses `hdiutil` on macOS, `losetup` on Linux)
- Injects configuration files

### ConfigurationInjection

- **NetworkManager**: Static IP or DHCP via systemd-resolved
- **Home Assistant**: Pre-seeds `configuration.yaml` with location, timezone, units
- **First-boot Status**: Systemd service that reports when HA is online

### First-Boot Flow

1. Raspberry Pi boots Home Assistant OS from micro-SD
2. NetworkManager connects using injected config (static or DHCP)
3. Home Assistant service starts
4. Status callback script (if enabled) polls HA API
5. Once HA is ready, sends webhook to monitoring server

## Troubleshooting

### Image won't mount on macOS

If `hdiutil attach` fails:

```bash
# Manually prepare image
gunzip haos-rpi5-*.img.gz
hdiutil attach haos-rpi5-*.img -mountpoint /tmp/ha-mount -nobrowse
```

### Network not working

- Verify NetworkManager config in `/etc/NetworkManager/system-connections/`
- Check if DHCP is preferred (leave `HA_STATIC_IP` empty)
- SSH to device: `ssh <username>@<ip>` using PASSWORD_HASH credentials

### Status callbacks not received

- Ensure `STATUS_IP` and `STATUS_PORT` are correct
- Check firewall allows connections on that port
- Verify HA service is actually running: `systemctl status ha-first-boot-status`

## Notes

- This does **not** install Docker or Frigate inside HA OS
- Home Assistant OS is purpose-built for running on Raspberry Pi hardware
- Network connectivity is required for initial setup and HA cloud integration
- For pure air-gapped deployment, leave `STATUS_IP` empty and configure manually via web UI
