# Ubuntu headless autoinstall, split into clean separate files

This bundle provides a streamlined workflow to build a custom Ubuntu Server ISO that automatically installs on a target machine, flash it to a USB drive, and optionally monitor the installation via callbacks.

## Workflow

### 1) Initialize & configure

```bash
cd ~/Downloads
unzip ubuntu-autoinstall-clean.zip -d ubuntu-autoinstall-clean
cd ubuntu-autoinstall-clean
./setup.sh
```

This script will:

- Install required Homebrew dependencies (macOS only)
- Generate a SHA-512 password hash and update `config.env`
- Check for the latest Ubuntu Server LTS ISO and optionally download it

At minimum, configure these in `config.env`:

- `ISO` — path to Ubuntu Server ISO
- `OUT` — output path for the custom ISO
- `HOSTNAME` — target machine hostname
- `USERNAME` — target machine username
- `PASSWORD_HASH` — SHA-512 hash (generated by `setup.sh`)
- `STATUS_IP` — local IP for receiving install callbacks (optional)
- `FLASH_DRIVE` — USB drive device path (e.g., `/dev/disk2` on macOS)
- `STATUS_PORT` — port for the status server (optional, default 8080)

### 2) Build & flash the ISO

```bash
./build-and-flash.sh
```

This script will:

- Extract the Ubuntu ISO into a temporary work directory
- Render `autoinstall.yaml` from the template using your config
- Patch GRUB to add autoinstall mode and shorten boot timeout
- Rebuild the ISO with modified files and checksums
- Flash the ISO to the configured USB drive using balena CLI
- Eject the USB drive safely on success

### 3) (Optional) Monitor the installation

In a separate terminal, start the status listener:

```bash
./broadcast.sh
```

This opens `http://localhost:8080/` to monitor installation progress. The status server receives callbacks from the target machine's cloud-init and Ubuntu autoinstall hooks, providing real-time visibility without needing to attach peripherals.

## Implementation

The implementation scripts are organized in the `lib/` directory:

- `lib/_01-install-deps.sh` — install Homebrew dependencies
- `lib/_02-generate-password-hash.sh` — generate a SHA-512 password hash
- `lib/_03-prepare-workdir.sh` — extract the Ubuntu ISO into a writable work tree
- `lib/_04-render-autoinstall.py` — render `autoinstall.yaml` from a template
- `lib/_05-patch-grub.py` — add `autoinstall` to the default GRUB boot entry and shorten timeout
- `lib/_06-rebuild-md5.py` — rebuild `md5sum.txt`
- `lib/_07-build-iso.sh` — repack the modified ISO
- `lib/_08-flash-image.sh` — flash the rebuilt ISO to the configured USB drive with balena CLI and eject it
- `lib/start-status-server.sh` — wrapper to launch the status server
- `lib/install-status-server.py` — receive install-status and first-boot callbacks

## Configuration

- `config.env` — your editable settings
- `config.env.example` — example configuration template
- `templates/autoinstall.template.yaml` — your installer template (substitutes `${KEY}` with values from `config.env`)

## Versioning

This project uses [Cocogitto](https://github.com/cocogitto/cocogitto) for semantic versioning and changelog generation based on [conventional commits](https://www.conventionalcommits.org/).

### Local version management

View the current version:
```bash
git describe --tags
```

Bump version (automatically updates `CHANGELOG.md` and creates a version commit):
```bash
cog bump --auto
```

View the changelog:
```bash
cat CHANGELOG.md
```

### CI/CD integration

The `cog bump` command can be easily integrated into GitHub Actions or other CI/CD platforms later. The workflow is designed to generate version tags and changelogs locally while remaining flexible for automation.

## Notes

- This workflow wipes the selected internal SSD and installs Ubuntu fresh.
- The template is set to reboot after install so cloud-init can send the first-boot callback.
- The status listener is only for progress visibility; it does not control the install.
- Dependencies are only installed on Intel-based macOS (ARM64 support may be added in the future).
